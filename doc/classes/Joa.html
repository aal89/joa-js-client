JOA Class

Defined in: D:\Documenten\Workspace\iwls\joa-js-client\src\Joa.js:95
Module: JOA

JOA is an object used to communicate with the backoffice of Munisense. This object will be able to construct a (syntactically) valid payload according to the ms-tech-141003-3 specification. This implementation only supports the MuniRPC version 2 protocol (JOA3).
There is no need to new this object as that is being done for you. Usage is through the JOA() object. Note: all requests made to the backoffice are made asynchronously.

  • [profileId] String optional

    an optional ZigBee specific field, if null is supplied the default 0xf100 will be used.

  • [clusterId] String optional

    clusters are an organizational unit of attributes of the same type. For example, all temperature related attributes are defined in clusterid: 1026 (0x0402). All cluster id's must be coordinated with Munisense before usage.

  • [isClusterSpecific] String optional

    attributes are the most specific fields defining a message. For example, the calibration value in the temperature cluster has an attributeid of 5 (0x005), has a unit in C (celsius) and has a data type int16s (0x29) and is presumed to be delivered with a scale factor of 0.01. Lists of definition are available from ZigBee specifications or vendor specific clusters can be defined coordinated with Munisense.

  • [commandId] String optional

    each attribute has a fixed data type. Sending this value is an indication how values submitted should be handled and must be consistent throughout the implementation. Data types are defined in the ZigBee specification.

  • [timestamp] String optional

    the timestamp is used to indicate the occurreence of a message. This value is a positive numerical value up to 48 bits in size indicating the number of milliseconds since 1970-01-01 00:00:00 UTC, not adjusting for daylight savings time or leap seconds.

    the value in an ASCII representation of the reporte values.

JOA.header

Object

Defined in D:\Documenten\Workspace\iwls\joa-js-client\src\Joa.js:38

The header object used to construct a valid header for a particular request.

gatewayIdentifier: The gateway identifier is a 32bit value formatted as an IP address. This IP address will not be the address of the sending node or the backoffice, but a virtual identifier assigned to this device.
The address will be an address in the private range defined in RFC1918.

protocolVersion: The HTTP POST content starts with a header indicating protocol and version. This contains the protocol-and-version string to be used in the header. Default is MuniRPCv2:. Note: this is a private property and cannot be changed

attribute: A header can also contain an optional comma separated list of value-attribute pairs.
attribute.vendor (string): The 'vendor' attribute is a string containing the assigned vendor name. A vendor field is required.
attribute.hash (boolean): The 'hash' attribute is the MD5 hash of the shared secret concatenated with the contents of the full HTTP POST, including the header without a hash attribute. This value is required when using security level 3 or 4. For this attribute a boolean should be given; true to hash the payload and false for not.
attribute.secret (string): The 'secret' attributes is the clear-text shared secret. This value is required for security level 2.
attribute.time (string): The 'time' attribute indicates a time request. This attribute does not need a attribute-value separator or a value. The reply messages will include a time reply. This time attribute works .
